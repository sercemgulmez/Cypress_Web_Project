## C) English Version

I published a QA automation portfolio project built with Cypress and TypeScript.

The focus is on demonstrating practical QA judgment alongside automation skills:

🔹 Risk-based test planning — deciding what to automate and what not to
🔹 Production-safe smoke and regression testing on public e-commerce flows
🔹 Manual-only boundary design — login, checkout, payment, OTP, and CAPTCHA intentionally excluded from automation
🔹 Fake data strategy — validating boundary behavior without real user data
🔹 Resilient selector strategy — ranked candidate arrays instead of brittle single selectors
🔹 Agent-assisted QA workflow — coding agent scaffolds architecture and docs, QA engineer owns safety decisions

Test automation is not only about writing scripts. It is also about knowing what should and should not be automated.

The project targets only public, read-only flows on a publicly accessible e-commerce site. No real logins, checkouts, payments, or account actions are performed. This is an independent portfolio/demo project and is not affiliated with any brand.
